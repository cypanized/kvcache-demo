Tiering the KV cache off the GPU

When the working set outgrows GPU HBM, prefill gets recomputed on every repeat request. LMCache tiers KV blocks to CPU RAM and external storage over NVIDIA NIXL — and repeat-request latency drops by up to 10×.

Where the KV blocks live

Tier 0 · GPU HBM
vLLM paged KV cache, capped at 74,880 tokens (~14 GB). Working set is 4× larger — this tier alone thrashes.
Tier 1 · CPU RAM
LMCache pinned-memory pool. 256-token chunks (48 MB each) moved over PCIe. Warm TTFT 0.17 s.
Tier 2 · External storage via NIXL
NVIDIA NIXL POSIX plugin → 1,500-slot file pool on NVMe (67 GB written). In production this is a VAST NFS / GDS mount. Warm TTFT 0.36 s.

Method & environment

  • Hardware: 1× NVIDIA A100 80GB PCIe (vast.ai, $1.03/hr), 24 vCPU, local NVMe.
  • Stack: vLLM 0.24.0 · LMCache 0.5.1 · NIXL (pip, POSIX plugin) · Qwen2.5-14B-Instruct, bf16.
  • Workload: 24 unique synthetic documents × ~12k tokens, 3 sequential passes (72 requests), streaming chat completions, max_tokens=32, temperature=0. TTFT = time to first streamed token, measured on-box (no WAN).
  • Pressure: --gpu-memory-utilization 0.55 caps the GPU KV cache at 74,880 tokens vs a ~290k-token working set, so the baseline's prefix cache evicts before any document repeats — the honest "GPU-only" failure mode at scale.
  • LMCache wiring: --kv-transfer-config '{"kv_connector":"LMCacheConnectorV1","kv_role":"kv_both"}'; NIXL tier via enable_nixl_storage, nixl_backend: POSIX, nixl_buffer_device: cpu, nixl_pool_size: 1500 (descriptor count — one 48 MB chunk file per slot, not bytes).
  • Gotchas we hit (so you don't have to): POSIX backend requires a CPU staging buffer; nixl_pool_size given in bytes allocates a few hundred GB of index and gets OOM-killed; the file pool holds one fd per slot, so raise ulimit -n. On a misconfigured backend LMCache degrades gracefully to recompute — correct answers at baseline speed.

Benchmarked 2026-07-18 on vast.ai instance 45214089. Total experiment cost ≈ $2.50 in GPU-hours.

The NVMe file pool stands in for any NIXL-reachable external target — an NFS/GDS mount on a VAST cluster behaves identically at the API level, with capacity decoupled from the GPU node.
